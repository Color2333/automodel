# Meshy-AutoModel 插件版本变更日志

## 版本 4.6.7 - 20260605

### 功能更新
- **新增分类标签**：状态标记新增 `nor-error` 与 `组合资产`，支持整体模型模式和仅选中对象模式。
- **新增分类目录**：导出目录新增 `NorError_{操作者}` 与 `ComboAsset_{操作者}`，进度恢复和旧输出清理逻辑同步支持新标签。

## 版本 4.6.6 - 20260604

### 功能更新
- **打标模式切换**：状态标记区域新增单按钮模式切换，可在「整体模型 / 仅选中对象」之间点击切换；原 good / bad / 存疑 / hard / 零件按钮会按当前模式执行。
- **选中对象单独分类导出**：在「仅选中对象」模式下，点击终态标签只导出当前选择；选中父级节点时会带上子对象，并继续自动补齐相关 Armature 以保留 rig/skin。
- **拆件编号递增**：选中对象分类导出使用 `{模型名}_{编号}` 命名，编号跨 Completed/Good/Bad/Questionable/Hard/Parts 目录递增，避免不同分类之间重名。

### 调整
- **选中对象模式不清旧分类**：单独分类导出不会删除该模型已导出的其他拆件，也不会自动切换下一项，便于同一模型内连续选择多个物品分别打标。

## 版本 4.6.5 - 20260521

### 修复
- **继续当前文件夹后旧 GLB 误删**：终态打标改为先成功导出/移动新文件，再清理旧分类副本；导出或移动失败时不会先删除已有成果。
- **Completed 移动覆盖保护**：从 `Completed_{操作者}` 移动到分类目录时，若目标已有同名文件会先临时备份，移动失败时恢复旧文件。

### 性能优化
- **断点保存降频**：自动拆分现场 `.blend` 保存增加 10 秒节流，快速编辑时不再频繁写完整场景。
- **按组导出减少卡顿**：按组导出时不再每个组保存一次拆分现场，改为整次导出成功后只保存一次；手动保存、Ctrl+S、切换输入源仍强制保存现场。

## 版本 4.6.4 - 20260515

### 新增
- **输入源切换**：侧栏新增输入源区域，会记录最近使用的源目录，可从下拉列表切换不同输入源。
- **按输入源恢复进度**：切换输入源时会先保存当前源目录进度，再加载目标目录自己的 `.progress/progress.json`、模型列表、当前索引、导出格式与历史。
- **输入源运行态保存**：Blender 重启后会保留最近输入源列表，并优先恢复上次有效输入源。

### 调整
- **输出目录随输入源切换**：切换到无历史进度的新输入源时，默认输出目录重置为该源目录的父目录，避免沿用上一个输入源的输出目录。

## 版本 4.6.3 - 20260514

### 修复
- **带骨骼 GLB 稳定导出**：导出前自动把选中/分组网格通过 Armature modifier 依赖的骨骼对象及其父级加入导出集合，避免“导出所有”或“按组导出”时 mesh 与 armature 分属不同顶级节点导致 skin 丢失。
- **显式启用 GLB skin 导出**：GLB 导出参数增加 `export_skins=True`，继续保持 `export_animations=False`，只保留骨骼绑定数据，不额外导出动作动画。

## 版本 4.6.2 - 20260511

### 功能更新
- **会话状态自动恢复**：保存源目录、输出目录、操作者、当前模型索引、导出格式和上次导出路径；Blender 重启后自动尝试恢复上次工作目录、加载 `progress.json` 并导入当前模型。
- **Ctrl+S 同步拆分现场**：挂接 Blender 保存前事件，人员按 `Ctrl+S` 时同步写入当前模型进度和 `.progress/scene_checkpoints/*.blend` 拆分现场，不再只保存普通 Blender 工程。
- **独立工程缓存写入**：checkpoint 改为直接写当前场景对象数据到模型专属 `.blend`，避免影响人员当前打开的主工程文件。

### 修复
- **崩溃后仍需重新选路径**：不再在打开文件后无条件清空源目录和操作者；若上次源目录仍存在，会自动恢复，不需要先手动重新导入路径。

## 版本 4.6.1 - 20260511

### 功能更新
- **拆分现场断点保存**：在自动保存进度时，为当前模型写入 `.progress/scene_checkpoints/*.blend` 场景快照；快速创建组、旋转、去 Alpha、清空组、导出、切换模型和手动保存进度都会同步保存当前拆分现场。
- **崩溃后恢复拆分进度**：重新设置源目录、加载进度或导入当前模型时，若存在该模型的场景快照，会优先恢复已拆分的父子层级和场景对象，减少崩溃后重新拆分。
- **终态后清理快照**：模型完成 good/bad/存疑/hard/零件等终态分类后，删除该模型的拆分快照，避免后续误恢复旧现场。

### 修复
- **加载进度后刷新覆盖状态**：刷新模型列表时保留已加载的状态、导出历史和上次导出状态，避免设置源目录后刚加载的 `progress.json` 被刷新流程重置。

## 版本 4.6.0 - 20260507

### 功能更新
- **整合 GLB/USDZ**：以最新 GLB 版 v4.5.5 为主线，新增导出格式选择，默认仍为 GLB；切换为 USDZ 后沿用同一套状态流转、目录与命名规则。
- **源目录支持 GLB/USDZ**：刷新模型列表时同时识别 `.glb` 与 `.usdz`；同名源文件同时存在时优先使用 GLB，避免同名模型重复进入列表。
- **USDZ 导入导出**：支持通过 Blender USD 导入器读取 USDZ，并通过 USD 导出器写出 `.usdz`，旧版 Blender 参数不兼容时自动回退。

### 修复与对齐
- **跨格式去重**：终态重新打标签、从 Completed 移动、直接终态导出时，按完整模型名匹配并清理 GLB/USDZ 旧输出，避免同名模型出现在不同标签目录。
- **可修复流程对齐 GLB 版**：可修复先导出到 `Completed_{操作者}`，再点击 good/bad/存疑/hard/零件移动到目标分类；命名仍为 `模型名.ext` 或 `模型名_序号.ext`。
- **导出失败保护**：GLB/USDZ 导出异常时恢复模型位置和选择状态，避免失败后污染当前场景。
- **USDZ导出兼容**：按当前 Blender 版本支持的 USD 导出参数动态调用，避免 `export_textures` / `export_relative_paths` 参数不兼容导致导出失败。

## 版本 4.5.5 - 20260423

### 修复
- **前缀同名模型误删/误移**：重写 `list_model_glbs_in_dir` 的文件匹配规则，修复当两个模型名存在前缀包含关系（例如 `k3dp_…PORTRAIT` 与 `k3dp_…PORTRAIT ONE PIECE`）时，后打标签的短名模型会误删/误移先前长名模型输出 GLB 的问题。具体改动：
  - 移除旧的 `glob.escape(key) + "*.glb"` 兜底分支——它会把任何以当前模型名开头的文件（包括同前缀的其他模型）都当成本模型的旧副本，是 `purge_terminal_exports_for_model`、`purge_completed_exports_for_model`、`move_completed_to_category` 三处共同的"前一个模型消失"根因。
  - 收紧 `{model}_` 后缀规则为 `{model}_{纯数字}.glb`，与 `export_objects` 生成的 `f"{model.name}_{group_index}"` 严格对齐；避免像 `foo` vs `foo_big_1.glb` 这种同前缀但非按组导出索引的文件被误判。
  - `move_completed_to_category` 的 LEP（last_export_path）兜底也对齐新规则：仅在 LEP 指向 Completed 内且文件名严格匹配时把该单文件作为源，不再按宽松规则重扫目录。
- 兼容性：`progress.json` 与各分类目录结构未变更；打标/导出/按组导出/从 Completed 移动的主流程行为保持不变。

## 版本 4.5.4 - 20260420

### 功能更新
- **可修复后可切换下一项**：标记为「可修复」后允许点击「下一项」继续给其他模型打标，与重新打终态标签后的自动跳转逻辑区分；「上一项」在可修复中仍禁用，侧栏提示同步更新。

### 性能优化
- **导入加速**：`import_scene.gltf` 新增 `import_pack_images=False`，大文件（>60 MB）不再将贴图打包进 .blend，显著减少导入时间和内存占用；新增 `guess_original_bind_pose=False`，跳过骨骼绑定姿态推算（插件会在导入后清掉空骨骼，此计算对静态模型无意义）。
- **导出加速**：`export_scene.gltf` 新增 `export_image_format='AUTO'`，保留贴图原始格式（JPEG/PNG），避免重新编码，对大文件导出提速明显。
- **导航加速**：移除切换模型时冗余的 `orphans_purge` 和 `view_all` 调用（`import_model` 内部已执行），减少每次切换的重复开销。
- **导出加速**：`export_objects` 中父级查找改用 `set` 替代 `list`，O(n²) → O(n)，对象数量多时（>几十个）避免重复线性扫描。

## 版本 4.5.3 - 20260401

### 修复
- **加载模型后残留**：此前用「全选 + 删除」清理场景，会漏删不可选对象、以及 glTF 导出器生成的 `glTF_not_exported` 集合内的物体等；现改为按 `scene.objects` 逐个移除数据块，并删除空子集合、执行孤儿数据清理，切换/重载模型后不应再留下棱角球等附加网格。

### 功能更新与改进
- **进度自动恢复**：`progress.json` 中每条模型除 `n/p/s` 外增加 `h`（导出历史）、`lx`（上次导出状态），保存/加载进度时一并读写；重开 Blender 或手动「加载进度」后可还原侧栏历史。若旧进度无 `h` 且为终态，仍尝试从分类目录下 `{模型名}.json` 侧车恢复。
- **导航**：「未标记」时允许「上一项」、禁止「下一项」；「可修复」时禁止「上一项/下一项」（须先导出 Completed 并完成终态分类）；侧栏按钮与快捷键行为一致。
- **重新打标签唯一输出**（延续 4.5.2）：标记终态前清理各终态目录及非移动路径下 Completed 中该模型旧 GLB，保证输出侧单份结果。

## 版本 4.5.2 - 20260331

### 功能更新与改进
- **修复与分类工作流**：
  - 标为「可修复」后先在场景中修复，再使用「按组导出 / 导出所有」将 GLB 写入 `Completed_{操作者}`。
  - 点 good / bad / 存疑 / hard（及零件）终态时，将 Completed 目录下本模型对应的 GLB（含 `模型名_N.glb` 多组）**移动**至分类目录，磁盘上仅保留分类侧一份文件。
  - 未进入「可修复」的初筛路径：直接点终态仍为「导出至分类目录 + 自动下一项」。
- **导出与输出目录**：
  - 任意已标记状态均可手动导出 GLB（仅「未标记」不可导出）；废除标记后自动写入 JSON 记录。
  - good / bad 对应输出文件夹为 `Good_{操作者}`、`Bad_{操作者}`（原 NoAction_/Unfixable_ 不再写入）；新增 **hard** 状态与 `Hard_{操作者}` 目录。
- **界面**：
  - 状态按钮文案：good、bad、存疑、hard、可修复；当前已为可修复时禁用再次点击「可修复」。
  - 导出区域增加说明：先导出到 Completed，再点分类。
- **兼容**：
  - `progress.json` 内状态枚举 ID 未变（仍为 NO_ACTION、UNFIXABLE 等）。
  - 加载进度时，对旧版 `NoAction_*` / `Unfixable_*` 下的侧栏 JSON 保留可选回退查找。
- **流程约束**：状态为「可修复」时禁止通过上一项/下一项（含 Ctrl+方向键）切换模型，须先完成导出与分类；侧栏导航按钮同步禁用并提示。
- **修复**：Completed 内查找待移动 GLB 时增加 Unicode NFC 归一化、放宽 `模型名_*` 后缀匹配，并在仍找不到时用「上次导出路径」回退，减少长文件名/含下划线模型名或系统编码差异导致的误报「未找到 GLB」。
- **修复**：「导出所有」原先误生成 `模型名_None.glb`（因传了 `group_name` 但未传 `group_index`），与移动逻辑期望的 `模型名.glb` 不一致；现仅在存在 `group_index` 时使用 `_序号` 后缀，合并导出为 `模型名.glb`。
- **调整**：good / bad 输出目录改为首字母大写 `Good_{操作者}`、`Bad_{操作者}`（与 `Completed_`、`Questionable_` 风格一致）；
- **重新打标签（唯一输出）**：标记终态前先从 Good/Bad/Questionable/Hard/Parts 删除该模型全部匹配 GLB；从非「可修复」路径导出终态前另清 `Completed` 中该模型陈旧 GLB。从「可修复」移动至终态时仍以 Completed 为源，不在移动前清空 Completed。从终态改回「可修复」时删除各终态目录中该模型文件。
